# Учебные C#-модули для code review

Репозиторий содержит C#-примеры для производственной практики по теме **«Ревьюирование программных модулей»**.

Проект сделан простым: это консольное приложение, которое можно открыть в Visual Studio, запустить и использовать для учебного ревью.

## Что внутри

```
Modules/
├── DiscountCalculator{Bad,Fixed}.cs    # 6 общих учебных модулей
├── UserRegistration{Bad,Fixed}.cs
├── OrderProcessor{Bad,Fixed}.cs
├── FileStorage{Bad,Fixed}.cs
├── Statistics{Bad,Fixed}.cs
├── Authorization{Bad,Fixed}.cs
└── Variants/                            # 30 индивидуальных вариантов
    ├── Variant01_LibraryBad.cs
    ├── Variant02_PharmacyBad.cs
    ├── ...
    └── Variant30_TheaterBad.cs
```

### 6 общих учебных модулей

У каждого модуля две версии: `Bad` (с типичными ошибками) и `Fixed` (исправленная для сравнения).

| Модуль | Плохая версия | Исправленная версия |
|--------|---------------|---------------------|
| Калькулятор скидок | `DiscountCalculatorBad.cs` | `DiscountCalculatorFixed.cs` |
| Регистрация пользователя | `UserRegistrationBad.cs` | `UserRegistrationFixed.cs` |
| Обработка заказов | `OrderProcessorBad.cs` | `OrderProcessorFixed.cs` |
| Чтение и запись файла | `FileStorageBad.cs` | `FileStorageFixed.cs` |
| Расчёт статистики | `StatisticsBad.cs` | `StatisticsFixed.cs` |
| Сервис авторизации | `AuthorizationBad.cs` | `AuthorizationFixed.cs` |

### 30 вариантов

Папка `Modules/Variants/` содержит код 30 предметных областей (по одной плохой версии на вариант): библиотека, аптека, школьный журнал, автосервис, гостиница, прокат автомобилей, кинотеатр, ресторан, складской учёт, парковка, магазин одежды, доставка еды, фитнес-клуб, стоматология, ветклиника, курьерская служба, турагентство, банкомат, прокат велосипедов, спортивные соревнования, инвентаризация, поликлиника, ЖКХ, дневник тренировок, автозаправка, турпоход, школьная столовая, прокат инструмента, зоопарк, театр.

## Какие ошибки нужно искать

В плохих версиях специально оставлены учебные ошибки по критериям А–И:

| Код | Критерий |
|:---:|----------|
| А | плохие имена переменных и методов |
| Б | слишком длинные методы и нарушение SRP |
| В | магические числа и строки |
| Г | дублирование |
| Д | отсутствие проверки входных данных |
| Е | отсутствие обработки исключений |
| Ж | смешение логики и ввода-вывода |
| З | неправильный выбор типов (`double` для денег и т. д.) |
| И | небезопасные решения (вывод пароля, ПИН в логах) |

## Требования

- .NET SDK 10.0 или новее;
- Visual Studio, Rider или VS Code;
- (опционально) SonarLint, StyleCop.Analyzers.

Проверить установленную версию .NET:

```bash
dotnet --version
```

## Как запустить

Запуск 6 общих учебных модулей:

```bash
dotnet run
```

Запуск конкретного варианта (1–30):

```bash
dotnet run -- 19    # вариант 19 «Прокат велосипедов»
dotnet run -- 1     # вариант 1 «Библиотека»
dotnet run -- 18    # вариант 18 «Банкомат»
```

## Как собрать проект

```bash
dotnet build
```

## Как открыть в Visual Studio

1. Откройте Visual Studio.
2. Выберите **Open a project or solution** и откройте `ReviewSamples.sln`.
3. В Solution Explorer откройте папку `Modules` для общих учебных модулей или `Modules/Variants` для индивидуальных вариантов.
4. Сначала изучите `*Bad.cs`, затем сравните с `*Fixed.cs`.

## Команды для проверки перед сдачей

```bash
dotnet build
dotnet run
dotnet format ReviewSamples.sln
```
